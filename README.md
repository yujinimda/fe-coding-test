# 🧑‍💻 프론트엔드 코딩테스트 스터디

프론트엔드 실무 기반 코딩테스트를 매일 1문제씩 풀어보는 스터디입니다.  
매일 자정 AI가 자동으로 Issues에 문제를 출제합니다.

## 진행 방식

1. 매일 **자정**에 Issues에 새 문제가 자동 등록됩니다
2. 각자 **브랜치**를 만들어서 문제를 풀고 **PR로 제출**합니다
3. PR에는 **풀이 설명**을 간단히 작성합니다 (접근 방식, 소요 시간 등)
4. **다음 날 자정**에 레퍼런스 정답이 댓글로 자동 공개됩니다
5. 머지는 다음 날 새 문제를 풀 때 본인이 직접 하면 됩니다

> 💡 **이 스터디의 목표는 매일 코딩하는 습관 만들기입니다.**
> - PR 리뷰는 필수가 아닙니다. 부담 갖지 마세요!
> - 취업 준비나 개인 사정으로 바쁘면 언제든 나가도 괜찮습니다.
> - 문제는 OpenAI API 토큰으로 자동 생성되며, **4월 말까지** 운영 예정입니다.

## 풀이 제출 방법

### 1. 레포 클론 (처음 1번만)

```bash
git clone https://github.com/yujinimda/fe-coding-test.git
cd fe-coding-test
```

### 2. 브랜치 생성

```bash
git checkout main
git pull origin main
git checkout -b day01/팀원1
```

> 브랜치 이름: `day{번호}/{이름}` (예: `day01/유진`, `day01/아름`)

### 3. 풀이 작성

```
problems/
  day01/
    팀원1.ts
  day02/
    팀원1.ts
    팀원2.ts
```

`problems/day{번호}/` 폴더에 `{이름}.ts` 파일로 풀이를 작성합니다.

### 4. 커밋 & 푸시

```bash
git add .
git commit -m "[Day01] 유저 프로필 타입 설계 - 유진"
git push origin day01/유진
```

### 5. PR 생성

GitHub에서 **Pull Request** 생성. PR 본문에 아래 내용을 포함해주세요:

- **접근 방식** (왜 이렇게 풀었는지)
- **고민한 부분**
- **소요 시간**

## 난이도 기준

| 난이도 | 기준 | 예시 |
|--------|------|------|
| 🟢 Easy | 개념 1개면 풀 수 있는 수준 | 타입 하나 정의, useCallback 적용 |
| 🟡 Medium | 개념 2~3개 조합 + 분석 필요 | 리렌더링 원인 찾기, 제네릭 조합 |
| 🔴 Hard | 설계 판단 + 엣지 케이스 + 성능 | 가상 스크롤, 타입 안전 이벤트 버스 |
